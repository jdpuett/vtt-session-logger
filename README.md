# VTT Session Logger

Converts Zoom VTT transcripts into polished, tagged session logs for [Campaign Logger](https://campaign-logger.com/) using the Claude API.

Built for tabletop RPG groups who record their sessions on Zoom and want well-formatted, searchable session logs without hours of manual writing.

## What It Does

1. Watches a folder for `.vtt` files (Zoom transcript exports)
2. Sends the transcript to Claude, which converts it into a narrative session log with Campaign Logger tags
3. Normalizes markdown for CL compatibility (`**bold**` → `__bold__`, `*italic*` → `_italic_`)
4. Splits the log into chunks that fit CL's 2400-character LogEntry limit, breaking at scene headings, sub-sections, or paragraph boundaries
5. Saves a local backup (full markdown, CL-format JSON, and individual chunks)
6. Optionally uploads directly to Campaign Logger via their API

## Requirements

- Python 3.10+
- [Anthropic Python SDK](https://github.com/anthropics/anthropic-sdk-python): `pip install anthropic`
- [Requests](https://docs.python-requests.org/): `pip install requests` (only needed for CL API upload)
- An [Anthropic API key](https://console.anthropic.com/)
- A [Campaign Logger](https://campaign-logger.com/) account (for API upload)

## Setup

### 1. Install dependencies

```bash
pip install anthropic requests
```

### 2. Set environment variables

```bash
# Required — Claude API
export ANTHROPIC_API_KEY=sk-ant-your-key-here

# Optional — Campaign Logger API (needed for auto-upload)
export CAMPAIGN_LOGGER_API_CLIENT=your-client-id
export CAMPAIGN_LOGGER_API_SECRET=your-client-secret
export CAMPAIGN_LOGGER_CAMPAIGN_ID=your-campaign-id
```

Generate your CL API client ID and secret from your Campaign Logger account's API Tokens page. The campaign ID can be found in Campaign Logger's URL or JSON exports.

### 3. Create your prompt file

Copy the example template and customize it for your campaign:

```bash
cp session_log_prompt.example.md session_log_prompt.md
```

Edit `session_log_prompt.md` with your campaign name, setting, player/character roster, and any campaign-specific instructions. This file is gitignored by default since it contains personal info.

## Usage

### Watch mode (default)

Drop a `.vtt` file into the script's directory and it converts automatically:

```bash
python vtt_to_session_log_v2.py
```

The script will prompt you for a session title before uploading to Campaign Logger.

### Chunk an existing markdown log

Process a session log that was already written or generated by a previous version of the script:

```bash
python vtt_to_session_log_v2.py --chunk session.md
```

### Chunk and upload

Same as above, but also uploads to Campaign Logger:

```bash
python vtt_to_session_log_v2.py --chunk-and-upload session.md
```

## Campaign Logger Tags

The prompt instructs Claude to apply CL tags on first mention of each entity:

| Tag | Usage | Example |
|-----|-------|---------|
| `@` | Characters & NPCs | `@"Aldric Vane"` |
| `#` | Locations & Places | `#"The Rusty Anchor"` |
| `^` | Groups & Organizations | `^"Dock Rats"` |
| `&` | Monsters & Creatures | `&"Troll"` |

## How Chunking Works

Campaign Logger limits LogEntry text to 2,500 characters. The script targets 2,400 to leave headroom for manual edits.

Chunks are split in order of preference:

1. At `## ` scene headings (the primary break point)
2. At `---` horizontal rules within a scene
3. At `### ` sub-section headings
4. At paragraph breaks (`\n\n`)
5. At line breaks (last resort)

Continuation chunks get a `(cont.)` suffix on the heading so they're identifiable in CL.

## Campaign Logger API

The script uses CL's v3 API with `api-client` and `api-secret` header authentication. On upload it:

1. Creates a new Log via `POST /logs` (prompts you for the title)
2. Posts each chunk as a LogEntry via `POST /log-entries`
3. Saves a manifest JSON with the results of each API call

If the API isn't configured, the script still works — it just skips the upload step and saves everything locally.
